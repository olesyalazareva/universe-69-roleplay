## Проблема

В ролевых играх и исследовательских сессиях сложно создать структурированную среду, которая одновременно:

- **Провоцирует спонтанное ролевое поведение**
- **Дает повторяемые условия** для сравнения
- **Собирает данные** в едином формате
- **Обходит психологические защиты** участников

**С чем сталкиваются исследователи и фасилитаторы:**

| Метод | Проблема |
| :--- | :--- |
| **Свободная ролевая игра** | Нет четких правил, участники "выпадают" из роли, данные несопоставимы |
| **Опросники и интервью** | Социальная желательность, участники говорят то, что "принято" |
| **Стандартизированные тесты** | Не дают спонтанного поведения, оторваны от реального контекста |
| **Наблюдение в естественной среде** | Данные сложно структурировать, нет повторяемости |

**В результате:** исследователи тратят время на подготовку, но получают несопоставимые данные, а участники не могут полностью погрузиться в ролевой опыт.

## Что предлагает UNIVERSE 69

UNIVERSE 69 — это **комплект материалов для проведения ролевых игр**, который решает эти проблемы через:

1. **Жесткую структуру UNO** — каждый ход привязан к правилам игры, участники не могут "съехать" с роли
2. **Карточные задания** — 54 задания разной сложности, которые провоцируют спонтанные реакции
3. **Роли (аватары)** — психологические архетипы для отыгрыша
4. **Протокол BIVREST** — встроенный механизм сбора данных с бинарной разметкой

**Ключевая идея:**

Вместо того чтобы спрашивать участника: _"Как ты себя чувствовал в роли?"_ — игра создает условия, в которых он **естественно проявляет ролевое поведение**. А группа через голосование фиксирует, было ли это поведение конгруэнтным.

Это **смещает фокус** с самоотчета на **наблюдаемое поведение** и дает данные, которые можно анализировать статистически.

## Что UNIVERSE 69 дает на самом деле

**Важно понимать:** UNIVERSE 69 не является терапевтической методикой или готовым AI-решением. Это **исследовательский инструмент**.

| Что UNIVERSE 69 дает | Что UNIVERSE 69 НЕ дает |
| :--- | :--- |
| Готовую структуру для ролевой игры | Готовый ответ на вопрос «хорошо ли сыграно?» |
| Экспериментальную среду с повторяемыми правилами | Клинически подтвержденную терапевтическую методику |
| Протокол сбора данных через BIVREST | Автоматическую генерацию датасетов |
| Возможность сравнивать сессии при фиксированных правилах | Готовое коммерческое AI-решение |

**UNIVERSE 69 — это инструмент для исследования ролевого поведения, групповой динамики и эмоциональных реакций в структурированной среде.**

Он полезен, если ваш исследовательский вопрос звучит так:

- «Как люди ведут себя в жестко ограниченных ролевых условиях?»
- «Какие типы заданий провоцируют конгруэнтное/неконгруэнтное поведение?»
- «Как группа воспринимает разные ролевые проявления?»
- «Можно ли собрать размеченные данные о ролевом поведении через игру?»

## Как работает игра

### Участники игры

| Роль | Описание |
| :--- | :--- |
| **Игроки** | 2–9 человек, 18+, каждый выбирает аватара (роль) |
| **Крупье** | Управляет колодой, следит за правилами, фиксирует результаты |
| **Группа** | Голосует после каждого действия через протокол BIVREST |

### Игровой цикл

**Выбор роли → Получение 6 карт и 6 фишек "Шанс" → Выкладка карты (ход по правилам UNO) → Отыгрыш задания с карты → Верификация группой (BIVREST) → Добор карт до 6**

### Компоненты игры

| Компонент | Описание |
| :--- | :--- |
| **Правила** | Полные правила игры, игровой цикл, механики |
| **Карты** | 54 карты с заданиями разной сложности |
| **Роли (аватары)** | Психологические архетипы для отыгрыша |
| **Сценарии** | 9 психологических сценариев для анализа игровых паттернов |
| **Протокол BIVREST** | Голосование "Верю/Не верю" и логирование |
| **Промпт для LLM** | Для соло-игры с AI |

### Матрица карт (по уровням сложности)

| Диапазон | Сложность | Примеры заданий |
| :--- | :--- | :--- |
| 2–5 | Низкая (разогрев) | Комплименты, коктейли, истории, пантомима |
| 6–9 | Средняя (раскрытие) | Презентация аватара, фантазии, объятия, подчинение |
| 10–Туз | Высокая (глубина) | Ощущения, эксперименты, раздевание, доминирование |
| Джокер | Максимальная (хаос) | Полное нарушение любых правил |

**Полный список 54 карт с заданиями — в `play/cards.md`**

## Структура данных

| Поле | Тип | Описание |
| :--- | :--- | :--- |
| `session_id` | string | Идентификатор сессии |
| `round` | integer | Номер раунда |

| `verdict` | string | «Верю» или «Не верю» |

| `empty_chair` | boolean | Был ли выход из роли |

## 🚀 Быстрый старт

### 1. Скачайте материалы

В этом репозитории есть всё необходимое для проведения игровой сессии.

### 2. Прочитайте ключевые документы

- **Правила игры** (`play/rules.md`) — детальные правила и игровой цикл
- **Колода карт** (`play/cards.md`) — 54 карты с заданиями
- **Роли (аватары)** (`play/avatars.md`) — реестр ролей (Лайт / Хард)
- **Матрица сценариев** (`design/mechanics.md`) — 9 психологических сценариев
- **Примеры логов** (`play/logs.md`) — как выглядят заполненные данные

### 3. Проведите сессию

#### Групповая игра (2–9 человек)

1. Соберите компанию (18+), выберите крупье
2. Распечатайте карты из `play/cards.md` и аватары из `play/avatars.md`
3. Раздайте каждому по **6 карт** и **6 фишек «Шанс»**
4. Каждый выбирает аватара
5. Следуйте правилам из `play/rules.md`
6. Фиксируйте результаты через BIVREST

#### Соло-игра (1 человек с LLM)

1. Откройте `play/solo-prompt.md`
2. Скопируйте промпт полностью
3. Вставьте в ChatGPT, DeepSeek или Claude первым сообщением
4. Играйте с AI-соперником

### 4. Анализируйте данные

Полученные логи дают возможность:

- **Количественного анализа** — частоты, распределения, динамика
- **Сравнительного анализа** — между разными ролями, игроками, сессиями
- **Обучения моделей** — если вы собираете данные для AI

## Интеграция с протоколом BIVREST

UNIVERSE 69 и **BIVREST** — два взаимодополняющих проекта:

- **UNIVERSE 69** создает контекст и провоцирует поведенческие реакции
- **BIVREST** оценивает эти реакции группой ("Верю/Не верю") и фиксирует результаты в лог

### Как BIVREST встроен в механику UNIVERSE 69

| Механика UNIVERSE 69 | Роль в BIVREST |
| :--- | :--- |
| **Карты с заданиями** | Генерируют информацию для оценки (стадия I) |
| **Аватары (роли)** | Задают контекст для оценки конгруэнтности |
| **Голосование «Верю / Не верю»** | Основной механизм верификации (стадия B) |
| **Фишки «Шанс»** | Ресурс, который теряется при «Не верю» |
| **«Пустой стул»** | Выход из роли при повторном «Не верю» |
| **Логирование** | Фиксация всех действий и голосов |

**Ссылка на проект:** [BIVREST на GitHub](https://github.com/olesyalazareva/bivrest-method)

## Ограничения

- **Игровая оценка ≠ объективная истина** — результат отражает восприятие группы, а не абсолютную реальность
- **Правила должны быть фиксированы** — если вы меняете правила, данные становятся несопоставимыми
- **Требуется несколько сессий** — для статистически значимых данных
- **Социальная желательность не исчезает** — она трансформируется в желание соответствовать ожиданиям группы
- **Это не сертифицированная терапевтическая методика** — инструмент для наблюдения и сбора данных, а не клинический протокол
