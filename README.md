## 1. Проблема и решение

### 🔴 Проблема
В ролевых играх и исследовательских сессиях сложно создать структурированную среду, которая одновременно:
* Провоцирует спонтанное ролевое поведение
* Даёт повторяемые условия для сравнения
* Собирает данные в едином формате
* Обходит психологические защиты участников

С чем сталкиваются исследователи, фасилитаторы и разработчики:

| Метод | Проблема |
| :--- | :--- |
| **Свободная ролевая игра** | Нет чётких правил, участники «выпадают» из роли, данные несопоставимы |
| **Опросники и интервью** | Социальная желательность, участники говорят то, что «принято» |
| **Стандартизированные тесты** | Не дают спонтанного поведения, оторваны от реального контекста |
| **Наблюдение в естественной среде** | Данные сложно структурировать, нет повторяемости |
| **Существующие Adult-приложения** | Поверхностный контент, быстрый отток пользователей (Churn Rate) |

> **В результате:** исследователи тратят время на подготовку, но получают несопоставимые данные, а разработчики не могут удержать аудиторию.

### 🟢 Решение
UNIVERSE 69 решает эти проблемы через:
* **Жёсткую структуру UNO** — каждый ход привязан к правилам игры, участники не могут «съехать» с роли
* **Карточные задания** — 54 задания разной сложности, провоцирующие спонтанные реакции
* **Роли (аватары)** — психологические архетипы для отыгрыша
* **Протокол BIVREST** — встроенный механизм сбора данных с бинарной разметкой
* **Визуально-сенсорный слой** — проективные триггеры, взламывающие социальные маски

> **Ключевая идея:** Вместо того чтобы спрашивать участника: *«Как ты себя чувствовал в роли?»* — игра создаёт условия, в которых он **естественно проявляет ролевое поведение**. А группа через голосование фиксирует, было ли это поведение конгруэнтным. Это смещает фокус с самоотчёта на **наблюдаемое поведение** и даёт данные, которые можно анализировать статистически и монетизировать через механику вовлечения.

## 2. Форматы продукта

Продукт разработан и протестирован как на физических носителях, так и в виде архитектуры для цифровой разработки:

* 🎲 **Физическая настольная игра «Казино 69»:**
  * 54 покерные карты с изображениями и заданиями, основанными на 9 базовых эротических фантазиях.
  * 2 карты «Джокер» — абсолютные катализаторы хаоса.
  * 54 физические фишки игровой валюты «Шанс».
  * Буклет правил и реестр аватаров для 2–9 участников.
  * *Применение:* Групповые психологические сессии, корпоративные тренинги, вечеринки 18+, исследовательские лаборатории.
* 📱 **B2C Web / Mobile App (MVP Architecture):**
  * Пошаговая State Machine на базе правил UNO.
  * Модуль быстрой социальной верификации BIVREST (`Верю / Не верю`).
  * In-App покупки тематических колод и премиум-аватаров.
  * Экономическая модель: Freemium + Battle Pass + Тематические DLC.
  * *Применение:* Цифровой продукт для массовой аудитории с высоким LTV и низким Churn Rate.
* 🤖 **AI Companion 18+ (Solo Mode):**
  * Готовый системный промпт, превращающий языковые модели (GPT-4o, Claude 3.5, DeepSeek R1/V3) в интерактивного ИИ-Крупье и игрового партнёра.
  * Возможность интеграции в чат-боты и голосовых ассистентов.
  * *Применение:* Персональные психологические практики, AI-компаньоны для взрослых, исследование поведения LLM в ролевых сценариях.

## 3. Игровая экономика и механики

### 📊 Базовые параметры
| Параметр | Описание |
| :--- | :--- |
| **Количество игроков** | 2–9 человек (или 1 игрок + ИИ-Крупье) |
| **Возрастное ограничение** | 18+ |
| **Вход в игру** | Единый взнос — 1 фишка «Шанс» с каждого игрока в Банк Казино |
| **Стартовый капитал** | 6 карт + 6 фишек «Шанс» на руках у каждого участника |
| **Выбор роли (Аватара)** | Бесплатно. При невозможности выбора — **1 «Шанс» в Банк** за выбор из реестра |
| **Победа** | Игрок с наибольшим количеством фишек «Шанс» к концу игры |

### 🔄 Механика хода (UNO-based)
1. **Определение первого хода:** Тот, у кого на руках младшая козырная карта.
2. **Выкладка карты:** Игрок кладёт карту той же масти ИЛИ того же номинала, что и предыдущая.
3. **Отыгрыш задания:** Игрок выполняет задание с карты **от лица своего аватара**.
4. **Верификация:** Группа/Крупье проводит проверку по алгоритму **BIVREST** (`Верю` / `Не верю`).
5. **Добор:** При успешном выполнении игрок добирает карту из колоды до 6 на руках.
6. **Смена хода:** Ход переходит к следующему игроку по часовой стрелке.

### 💰 Штрафная сетка (Экономика вовлечения)
| Нарушение | Штраф | Экономический смысл |
| :--- | :--- | :--- |
| **Отказ / невозможность выполнить задание** | 1 «Шанс» в Банк | Стимулирует выход из зоны комфорта |
| **Выход из роли / неискренность (вердикт «Не верю»)** | 1 «Шанс» в Банк | Наказывает за поверхностный отыгрыш |
| **Нет карты для покрытия** | Покупка карты у соперников или Крупье за **1 «Шанс»** | Поддерживает циркуляцию валюты (пропустить ход нельзя!) |

> **Экономический эффект:** Постоянная циркуляция «Шансов» создаёт напряжение, вовлекает игроков и формирует эмоциональную привязку к игровому процессу.
